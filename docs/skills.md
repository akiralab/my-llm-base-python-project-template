# SKILL運用ガイド

## 1. 目的

繰り返し発生するオペレーションをSKILL化し、作業品質と再現性を高める。

## 2. SKILL登録が必要なケース

- 同種作業を月2回以上繰り返す場合
- 手順漏れが発生しやすい定型業務
- 属人化を避けたい分析・更新・配布作業

## 3. SKILL登録時の必須記載

- 用途（何のための作業か）
- 入力（必要ファイル・パラメータ）
- 出力（成果物と保存先）
- 実行手順
- 失敗時の復旧手順
- 注意事項（前提条件・権限）

## 4. レビューと承認

- SKILL追加・更新はPRで管理する。
- 最低1名の人間レビュー承認後に有効化する。

## 5. 保守運用

- 依存ライブラリやデータ仕様の変更時にSKILLを更新する。
- 半年に1回、実運用との乖離を棚卸しする。
