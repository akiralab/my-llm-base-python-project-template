# LLMコンテキスト運用ルール

## 1. 目的

人間とAIエージェント間で、開発コンテキストを再利用可能にする。

## 2. 保存場所

- 本ディレクトリ配下（`docs/llm-context/`）に保存する。

## 3. 命名規則

- `YYYYMMDD_topic_owner.md`
- 例: `20260222_data_split_policy_team-a.md`

## 4. 記載ルール

- 背景
- 決定事項
- 未決事項
- 次アクション

## 5. セキュリティ

- APIキー、個人情報、機密情報は記載しない。
- 外部共有不可情報は要約のみ記載する。

## 6. 承認ルール

- 重要な運用ルール変更は、人間レビュー承認を経て反映する。
